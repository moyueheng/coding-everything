# 上游更新报告模板

按这个结构写 `docs/upstream-updates/` 下的报告。可以删减无变化的部分，但不要改成流水账。

## 标题

```md
# 上游仓库更新报告（YYYY-MM-DD）
```

## 1. 更新范围

```md
## 更新范围

- 更新时间：YYYY-MM-DD
- 更新方式：`git submodule update --remote` / 单仓库 `git pull --ff-only`
- 覆盖仓库：`upstream/...`
- 证据来源：git diff、git log、上游 README/docs/具体文件
```

## 2. 变更概览

```md
## 变更概览

| 仓库 | 旧 SHA | 新 SHA | commit 数 | 结论 |
|------|--------|--------|-----------|------|
| `upstream/superpowers` | `abc1234` | `def5678` | 6 | 新增 2 个可复用 workflow，值得跟进 |
```
```

`结论` 要短，必须可被后文证明。

## 3. 各仓库详情

```md
## upstream/superpowers

### 本次更新包含什么

- `2026-03-01` `abcdef1` Add ...
- `2026-03-03` `bcdefg2` Fix ...

### 具体变化

- 新增：
- 调整：
- 修复：

### 与本项目的关系

- 可以直接复用：
- 需要手动适配：
- 暂时无需跟进：
```
```

如果 commit 很多，先按主题归类，再列代表性 commit，不要无脑贴完整列表。

## 4. 值得关注

```md
## 值得关注

### 1. 标题

- 证据：对应 commit / 文件 / README 段落
- 为什么重要：说明它会怎样影响本项目
- 建议动作：同步、借鉴、观察、暂不处理
```
```

只有在证据充分时才写“值得关注”。

## 5. 后续动作

```md
## 后续动作

- [ ] 是否同步到 `kimi/` 或 `opencode/`
- [ ] 是否补充测试 / 验证命令
- [ ] 是否更新 `AGENTS.md` / `README.md`
```
